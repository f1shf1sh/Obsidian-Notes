# 硬件层
所有设备通过IBMC管理，包括操作系统安装，硬盘划分等
## 硬盘
服务器硬盘分为物理盘和逻辑盘，在添加物理盘且IBMC可以识别后，将物理盘编辑成逻辑盘
## 风扇问题
风扇可以通过IBMC调节，默认的模式比较激进，为了防止器件温度过高。
如果PCI上面连接了设备，可能会导致风扇一直比较激进。
# 推理卡程序安装
## 几个安装包的关系
第 1 层：硬件使能层
	这是“让卡能被系统正确识别和工作”的最底层。
  - Ascend-hdk-310p-npu-driver_*.run
    作用：安装 NPU 驱动
  - Ascend-hdk-310p-npu-firmware_*.run
    作用：安装 NPU 固件
  - Ascend-hdk-310p-mcu_*.zip
    作用：升级卡上的 MCU 管理固件
第 2 层：运行时/开发层
	这是“让应用、推理程序、测试工具能调用 NPU”的用户态环境。
  - Ascend-cann-toolkit_8.5.0_linux-aarch64.run
    作用：装 CANN Toolkit
    包含运行时、编译器、ACL、工具链等核心用户态环境
  - Ascend-cann-310p-ops_8.5.0_linux-aarch64.run
    作用：装 310P 对应的算子包
    没它很多模型跑不起来，或者性能不对
  - Ascend-cann_8.5.0_linux-aarch64.run
    作用：这是一个“CANN 总包/组合包”
    本质上还是第 2 层，只是打包方式更大、更笼统
第 3 层：测试/管理工具层
	这是“拿来测性能、看状态、做管理”的工具。
  - ais_bench
    作用：跑推理性能测试
  - ascend-dmi
    作用：做算力/带宽/状态基线测试
  - npu-smi
    作用：看卡状态、功耗、温度、版本

推理卡下载链接：https://www.hiascend.com/hardware/firmware-drivers/community?product=2&model=17&cann=8.5.0&driver=Ascend+HDK+25.5.1
推理卡固件和驱动的安装
```sh
# 内核源码安装
dnf install -y make dkms gcc kernel-headers-$(uname -r) kernel-devel-$(uname -r)
ls -ld /usr/src/kernels/$(uname -r)
ls -ld /lib/modules/$(uname -r)/build

groupadd HwHiAiUser
useradd -g HwHiAiUser -d /home/HwHiAiUser -m HwHiAiUser -s /bin/bash

chmod +x Ascend-hdk-310p-npu-driver_25.5.1_linux-aarch64.run
./Ascend-hdk-310p-npu-driver_25.5.1_linux-aarch64.run \
    --full \
    --install-username=HwHiAiUser \
    --install-usergroup=HwHiAiUser
  
chmod +x Ascend-hdk-310p-npu-firmware_*.run
./Ascend-hdk-310p-npu-firmware_*.run --full

reboot
npu-smi info
```
安装MCU
```sh
# 1. 解压 MCU 包
  cd /opt
  unzip Ascend-hdk-310p-mcu_*.zip
  ls *.hpm

# 2. 查卡的 NPU ID
  npu-smi info -l

# 3. 对每张卡逐个升级，不能并发
# 假设先升 NPU ID 0：
  npu-smi upgrade -t mcu -i 0 -f /opt/Ascend-hdk-310p-mcu_xxx.hpm
  
  #如果看到类似 need active mcu，继续执行生效：
  npu-smi upgrade -a mcu -i 0
  
# 4. 等 30 秒后查版本，确认成功：
  npu-smi upgrade -b mcu -i 0

# 5. 再按同样方法升 1、2、3
  npu-smi upgrade -t mcu -i 1 -f /opt/Ascend-hdk-310p-mcu_xxx.hpm
  npu-smi upgrade -a mcu -i 1
  npu-smi upgrade -b mcu -i 1
```
注意
  - MCU 升级必须逐张卡做，不能并发。
  - 升级过程中不要关机、重启。
  - MCU 升级过程中，以及生效后 2 分钟内，不要再对这张卡做 MCU 操作。
  - 如果升级的是 MCU 3.3.4 及以上，官方要求 npu-smi >= 22.0.3。
  - `npu-smi upgrade -b mcu -i <id>` 显示出新版本号，才算成功。

# 问题
1. 为什么安装驱动需要下载与内核版本相关的源码？
2. 为什么安装驱动需要添加额外的用户？
```sh
	• HwHiAiUser 是昇腾驱动默认推荐的“运行用户”，不是“安装用户”。
  简单说：
  - root：负责安装 driver / firmware / MCU
  - HwHiAiUser：负责驱动运行时相关的权限归属

  它的作用主要是这几个：
  - 给昇腾驱动和相关运行时组件一个非 root 的默认属主
  - 让后续 CANN、推理程序、管理服务有稳定的权限边界
  - 避免把整套昇腾运行环境都放在 root 身份下运行

  所以它更像是一个“专门给 Ascend/NPU 运行环境准备的系统账户”。

  你可以这样理解：

  - 不是拿它来登录系统干活的
  - 主要是给驱动安装器、运行目录、服务进程、权限控制用的

  为什么官方推荐它：
  - 比直接用 root 更安全
  - 多用户环境更好管权限
  - 后面装 CANN、跑业务、跑容器时，文件属主和权限不容易乱

  如果不用它，也能装，但你要在驱动安装时显式指定：
  --install-username=root --install-usergroup=root --install-for-all

  这种方式的问题是：
  - 官方提示有安全风险
  - KMSAgent 服务不可用
  - 后续运行环境更容易全部变成 root 权限
	```
